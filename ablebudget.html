ableBudget

A personal budgeting dashboard that connects to my real bank accounts through Plaid (read-only) and pulls balances and categorized transactions into a local database to track budget vs. actual. It runs entirely on my machine, so the access token and the data never leave it.

STATUS: BUILT & RUNNING LOCALLY · LIVE BANK DATA OVER PLAID

What it is

Link a bank, pull balances and categorized transactions into a local SQLite database, set monthly budgets per category, and watch budget vs. actual update. A month stepper in the nav drives every page: dashboard, transactions, and budgets.

The scope stays narrow on purpose: single-user, no login, no cloud, no money movement. It only uses Plaid's transactions product, and only to read.

Why build it

Most budgeting apps want your bank login on their servers and your spending history in their cloud. I wanted the reverse: my data on my own machine, under my control.

It is also the first software project I have built that handles money, so most of the care went into the privacy and trust model rather than the charts.

How it stays private

This touches real financial data, so the trust model came first. The approach is straightforward: secrets stay on the backend, and the backend stays on localhost.

What it taught

The access token stays on the backend. It lives only on the server and in the local database, and is never sent back in an API response. The browser only ever handles the throwaway public_token.

Watch the sign on every amount. In Plaid a positive amount means money leaving the account. Every budget calculation depends on getting that one rule right.

A manual edit should outlast the next sync. Re-categorizing a transaction writes a category override that a later sync will not overwrite.

A local app has no webhook. With no public URL there is nothing for Plaid to call back, so on-demand cursor-based "Sync now" handles updates instead.
